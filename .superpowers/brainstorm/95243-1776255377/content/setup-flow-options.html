Which first-run setup structure feels right?

All three assume: / is blocked until an org exists, /setup is public on first run, the wizard creates the first admin account, and the flow includes guided .env instructions.

A. Linear Gate
Setup progress
1. Environment checklist
2. Auth strategy
3. Create first admin
4. Create organization
5. Finish + go to login

Linear Gate

Strict step-by-step onboarding. Great for reducing confusion and guaranteeing order, but the most guided and least flexible.

B. Setup Launcher
Choose your path
Deployment guide
Auth strategy
Create admin account
Organization details
Progress checklist stays visible

Setup Launcher

Starts with a choice-oriented setup home, then moves into short focused tasks. Best balance of guidance, flexibility, and “prompted with options.”

C. Operations Console
Checklist
Env
Auth
Org
Done
Status panels
Current task detail
Docs / commands / next steps

Operations Console

Feels powerful and admin-heavy, with lots of context visible at once. Strong for self-hosters, but denser and more complex for first-run onboarding.

Recommended

  • B. Setup Launcher best matches your “prompted with options, then instructions” requirement.
  • Keeps /setup feeling intentional rather than just a long form.
  • Makes it easy later to add optional setup tracks like Google sign-in or Workspace.

Watch-outs

  • A can feel too rigid if the admin already knows what they need.
  • C risks feeling like a dashboard before the app is actually ready for use.
  • B needs clear completion states so it does not feel “choose-your-own-adventure” in a bad way.
